Part 1: Fun with Filters

1.1 Convolutions from Scratch

I implemented 2D convolution from scratch using nested loops to apply a 9x9 box filter kernel. The implementation handles boundaries by checking if pixel coordinates are within the image bounds before accumulating values (zero-padding approach). This means pixels at the boundary only use valid neighbors, which can result in darker edges.

# Create 9x9 box filter kernel kernel = np.ones((9, 9), dtype=np.float32) / 81.0 # Load and prepare image image_array = cv2.imread('Alex ID.jpg', cv2.IMREAD_GRAYSCALE) image_array = cv2.resize(image_array, (300, 300)).astype(np.float32) # Get dimensions image_height, image_width = image_array.shape kernel_height, kernel_width = kernel.shape pad_height = kernel_height // 2 pad_width = kernel_width // 2 # Initialize output output_image = np.zeros_like(image_array).astype(np.float32) # Perform convolution with nested loops for i in range(image_height): for j in range(image_width): for m in range(kernel_height): for n in range(kernel_width): # Calculate source pixel coordinates image_row = i - pad_height + m image_col = j - pad_width + n # Zero-padding: only use pixels within bounds if 0 <= image_row < image_height and 0 <= image_col < image_width: output_image[i, j] += image_array[image_row, image_col] * kernel[m, n] # Compare with scipy implementation from scipy.signal import convolve2d output_image2 = convolve2d(image_array, kernel, mode='same')

Comparison with scipy.signal.convolve2d:

  • Runtime: My implementation using nested Python loops is significantly slower (O(N²M²) where N is image size, M is kernel size) compared to scipy's optimized implementation which uses FFT-based convolution for larger kernels.
  • Boundary Handling: My implementation uses zero-padding (ignoring out-of-bounds pixels), while scipy's 'same' mode uses different boundary extension methods by default. This causes slight differences at image borders visible in the comparison images.

Threshold chosen: 35.0. This value strikes a balance between detecting meaningful edges and suppressing noise. A lower threshold would capture more edge detail but introduce significant noise from texture and minor intensity variations. Differences: The DoG filter produces significantly cleaner edge detection compared to the finite difference operator. By applying Gaussian smoothing before computing derivatives, the DoG approach effectively suppresses high-frequency noise while preserving meaningful edge information. This results in more continuous, well-defined contours of the person, camera, and other objects in the scene. The edges appear smoother and more coherent, making it easier to identify the overall structure and outlines of different elements in the image, whereas the finite difference approach produces more fragmented edges with visible noise artifacts.

Part 2: Fun with Frequencies

2.1 Image Sharpening

Image sharpening is achieved through the unsharp masking technique, which enhances edge detail by manipulating frequency components. The process works as follows: First, we apply a Gaussian low-pass filter to create a blurred version of the image, isolating the low frequencies. Next, we subtract this blurred image from the original to extract the high-frequency components (edges and fine details). Finally, we amplify these high frequencies by a factor α and add them back to the original image. This makes edges more pronounced and the image appear sharper. The formula is: sharpened = original + α × (original - blurred).

Most Important Thing Learned

Technical Insight: The most important technical concept I learned is that convolutions can be used to compute discrete derivatives. This fundamentally changed how I view CNNs. I now understand that CNNs are capable of learning to find derivatives and progressively higher-order derivatives as you go deeper into the network layers. This perspective helps explain why deeper layers can capture increasingly complex and abstract features in images.

Project Management Lesson: The most valuable non-technical lesson I learned is the importance of creating a scaffold of all deliverables before diving into the actual project implementation. By first understanding exactly what needs to be produced, what images, visualizations, and results are required, I can write code that directly targets those specific outputs. This approach prevents wasted effort and ensures that all project requirements are met systematically.
